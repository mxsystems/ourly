  1. User Roles and Permissions

    User Roles

    The purpose of user roles is to give an employee or contractor the ability to track time spent on any given project in real time.

    This user roles should be granted for individuals that need the ability to record time spent on individual task as well as track overall individual time spent on one or more projects over the extent of their employment.

    Specific user roles can create time entries for their specific task completed as well as view multiple displays of the time they have previously charged.

    different user roles are given the ability to export time charged to a .csv for their reference or use otherwise.

    Our.ly timetracker contains the following user roles:

    Authenticated User
    Project Manager
    Administrator

    Authenticated User

    Authenticated users are the basic users that can login and create time entries. They can create time entries for general purposes as well as for projects that they have been assigned to. Additionally, they will be able to view and export all of their time entries.

    Project Manager

    The purpose of this user role is to give an employee or contractor the ability to create and manage key project information for one or more projects as well as all of the capabilities available to a general user.

    Specifically this user role has the additional ability of creating a project, managing the general users assigned to the project, and tracking key statistical information gathered during the project including time and expenses utilized.

    This user has the ability to view the hourly rates of General Users assigned to their project and use that information to track and manage a project budget. Furthermore this user has the ability to manage what project and categories a general user can charge time to for their specific projects.

    This user has the ability to send project updates to General Users and assign task for them to accomplish in within the system.

    Administrator

    The purpose of this user role is to give an employer or employee the ability to manage multiple employees, clients, and projects gathering critical time and expenses data as well as all of the capabilities available to a Project Manager.

    Specifically this user role has the additional ability of creating and managing a client profile in the overall management system. From here this user has the ability to create a project for these clients and assign a Project Manager and General User(s) to work on the project.

    This User role is also responsible for managing the employees and contractors with access to this management system. This user can manage what projects Project Managers are managing and what projects General users have the ability to contribute to.

    This User role is also responsible for managing categories that time can be charged to. A time category is responsible for breaking a project or project team into differentiable areas for better tracking of time and expenses spent on specific areas of the project. For instance, categories can be created for tracking time and expenses for the design and development aspects of the project. From here this user classification can glean information on where a majority of time and money is being used over one or many projects.

    This user role is provided an overview of all projects new and old listed in a hierarchal structure that favors projects with the most recent activity. From here the user can view information on time that is being charged by users to specific projects as well as overall time charged during a specific time frame.

    This section will also provide live financial data relevant to project budgets and current totals of time charged to date to the projects.

  2. Creating and Managing a Users

    The following categories are available for entering user information:

    * Name – The Name field is used to display the prefix, First Name, and Last Name of the user.

    * User Name - The username field is used for a Unique username between 5 and 14 characters long.

    * Password – The password field is for a unique login password that must follow strict password restrictions.

    * Email – This email field will be used to store an email address for all correspondences pertaining to the use of this system.

    Phone – The phone number field is used for optional correspondence between users and system administrators.

    Hourly Rate – The hourly rate field is used to calculate budgets and track project expenses. This should be displayed in $ #,###.## format.

    * Status

    An Active user will have access to the system and will be able to log in and make changes based on their permissions.
    A Blocked user will be blocked from logging in to the system but all of their prior entered data will remain in place.

    * Roles

    General User
    Project Manager
    Administrator

    *See user definitions for an explanation of the available features to each user group.

    * Time Zone – The Timezone field is used to track local time of day for task completion and rate calculation.

  3. Creating and Managing a Client

    * Name – The Name field will be the main reference name for all aspects of this project. This can be used to store the parent name of the company as well as the project manager/team on the client side for use with a sub client of a larger client.

    Description – The description field is used to define the specific variables and the overall client information for the specified client.

    URL – The URL field is used to store the clients website information.

    Image – The image field is used to store a client logo or picture for easier identification of project associations. This file has a default image capacity of 2MB and an acceptable file format of png, gif, jpg, jpeg.

    Parent Relation - The parent relation field is used to specify a larger organization/client that this client is associated with. The default selection for this is a to be a top-level client.

  4. Creating and Managing a Project

    * Title – The title field will be the main reference name for all aspects of this project.

    Project Description – The project description field is used to define the project specific variables and the overall goal and direction of the project. As well as provide general providing general contact information for the client side manager of the project and other key information.

    Project Hours – The Project Hours field is used to declare the total number of hours to be spent on this project.

    Billing Rate – The billing rate field is used to store the hourly breakdown of what the client is being charged for this specific project.

    Total Budget – This field encompasses the total project hours times the billing rate plus additional expenses that have been planned for.

    Client – The Client field is a direct link to the available clients in the system and their contact information.

    Team Members – The Team Member field is used to assign users to the project from a list of available users.

  5. Creating and Managing a Category

    * Name – The Name field will be the main reference name for all aspects of this category. This can be used to store the overall name of a category or the name of a sub-category.

    Description – The description field is used to define the specific task or functions that should be done when using this category.

    Parent Relation - The parent relation field is used to specify a larger overall category that this specific category falls under. (i.e. Project Proposals could be a subcategory of Administrative) The default selection for this is a to be a top-level category.

  6. Time Entry

    * Date – The Date field will be used to specify the date that the work was completed on. This date can be today or any date in the past. This field can be locked down to only allow time entry for a specified period. For example the user can only charge time for the current calendar week.

    * Time Spent – The time spent field is used to record the amount of time spent for this specific task and description combination. This number should be expressed in a quarter hour two decimal format. (i.e. 1.25, 1.50, or 1.75)

    Note – The note field is used to write a description of the work or task accomplished during this time period.

    Category – The category field is used to describe what part of a project or what classification the work performed would fall under.

    Project - The project field is used to specify the project that the task or work was completed for.
